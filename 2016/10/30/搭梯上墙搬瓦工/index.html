搭梯上墙搬瓦工

发表于 2016-10-30

搭梯上墙搬瓦工

在使用搬瓦工之前我一直都是用的免费的 XX-Net 进行墙外访问，平时也就用用 google 搜索和邮箱，因此对带宽需求不大。但是最近发现 XX-Net 的访问速度变得越来越慢了，同时又有人推荐了搬瓦工这个相对廉价的 VPS （我先购买了试用了一个月，不到 30 RMB ，事实上还有更加廉价的，但是我买的那会已经没了），因此也就有了后来我弃 XX-Net 转搬瓦工之举。这篇博客记录了我通过搬瓦工搭建 SS （Shadowsocks 简称）的过程，希望能对后来的读者能有所帮助。

流程

  • 购买 VPS
  • 安装 Shadowsocks
  • 通过 FinalSpeed 加速
  • 实现过滤

操作系统

Centos6 x86

一、购买 VPS

注册购买 VPS （注意这里可能需要先翻墙才能访问）。

https://bandwagonhost.com/

关于如何购买以及购买之后如何对 VPS 进行管理，在搬瓦工中文站新手教程中有详细的介绍，这里不再赘述。

需要注意的一点是，由于之后需要使用 FinalSpeed 进行加速，所以在购买 VPS 的时候要求内存至少得是 128MB 的，否则 FinalSpeed 跑不起来，科学上网的网速也会很尴尬。

二、安装 Shadowsocks

VPS 端的 Shadowsocks 安装

首先通过 SSH 远程登录 VPS 安装 ShadowsockR 。

ShadowsocksR 是 Shadowsocks 的一个分支，据说比较不错，通过 Shell 脚本来进行安装。ShadowsocksR 默认采用 aes-256-cfb 加密方式，如果想要在手机端也使用 SS 科学上网的话，建议改为 chacha20 加密算法，该算法针对手机以及路由器有特殊的优化。

安装过程会提示配置密码（默认 teddysun.com）与端口号（默认 8989），可以使用默认也可以按个人需求进行配置。这里的端口号与密码就是用于客户端连接使用的。

当然还有种更加简单的方法，就是进入搬瓦工 VPS 后台使用官方提供的 SS ，但是这一做法并不推荐。

到这一步操作之后，只要之前的配置没有问题就可以科学上网了。但是比较遗憾的一点是目前这样科学上网网速比较慢，因此我们还要使用 FinalSpeed 进行加速。

三、通过 FinalSpeed 加速

FinalSpeed 是一个高速双边加速软件，可加速所有基于 TCP 协议和 UDP 协议的网络服务。但是有个缺点就是占内存相对较高，不过通过来说有个 128 MB 的内存就已经足够了。

四、实现过滤

到目前为止似乎一切都已经很完美了，但事实上还差一步。就是目前我们本地主机任何的 HTTP 请求都会通过代理服务器进行转发，这对于访问国外网站不成问题，然而一旦我们要访问国内网站的时候就会变得出奇的慢。

所以为了解决这一问题，我们还需要设置过滤规则来判断哪些请求地址需要通过代理转发哪些又不需要。由于设置的方式比较多，这里我只简单介绍几样。

使用 Shadowsocks 自带的在线 PAC 智能代理

Shadowsocks 的系统代理模型有两种：

  • PAC 模式
  • 全局模式

通常我们并不使用全局模式，所以可以使用 Shadowsocks 自带的在线 PAC。

使用 GFWList 设置本地 PAC 代理文件

Shadowsocks 的在线 PAC 模式毕竟还不够智能，而且覆盖的也不够全，另外也不能结合自己实际情况进行定制，所以我们还可以使用本地 PAC。

该模式下 Shadowsocks 会从目录中的 pac.txt 文件中读取过滤规则，因此我们只要把过滤规则写入该文件就能实现智能代理了。可是这个规则不是人人都懂得怎么写，而且也不是人人都想这么折腾去学习怎么写，所以我推荐一个更加人性化方法，我们将通过 GFWList 来更新本地 PAC 。

步骤为： PAC -> 更新本地 PAC 为 GFWList 。这样一来本地的 PAC 文件将自动根据 GFWList.txt 生成过滤规则。

使用 Chrome + SwitchyOmega 实现过滤

使用 Chrome 浏览器的用户还可以安装 SwitchyOmega 插件来实现智能过滤是我最推荐的一种的方式，也是我目前在使用的方案，使用该方案不仅可以使用现成的 GFWList 还能在平时浏览中手动添加过滤规则，极为方便。
