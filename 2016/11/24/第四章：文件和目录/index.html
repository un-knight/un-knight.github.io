第四章：文件和目录

发表于 2016-11-24   |   分类于 Linux   |     |   阅读次数

重要概念

文件类型

UNIX 系统的文件类型包括以下几种：

  1. 普通文件 (regular file) 。这种文件包含了某种形式的数据。至于这种数据是文本还是二进制数据对于 UNIX 内核而言并无区别。对普通文件内容的解释由该文件的应用程序进行。

补充：二进制可执行文件是一个值得注意的例外。为了执行程序，内核必须理解其格式。所有二进制可执行文件都遵循一种格式，这种格式使内核能够确定程序文本和数据的加载位置。

  1. 目录文件 (directory file) 。这种文件包含了其他文件的名字以及指向与这些文件有关信息的指针。对一个目录文件具有读权限的任一进程都可以读该目录的内容，但只有内核可以直接写目录文件。进程必须使用特定的函数才能更改目录。
  2. 块特殊文件 (block special file) 。这种文件类型提供对设备（例如磁盘）带缓冲的访问，每次访问以固定长度为单位进行。
  3. 字符特殊文件 (character special file) 。这种文件类型提供对设备不带缓冲的访问，每次访问长度可变。系统中的所有设备要么是字符特殊文件，要么是块特殊文件。
  4. FIFO 。这种类型文件用于进程间通信，有时也将其称为命令管道 (named pipe) 。
  5. 套接字 (socket) 。这种文件类型用于进程间的网络通信。套接字也可用于在一台宿主机上进程之间的非网络通信。
  6. 符号链接 (symbolic link) 。这种文件类型指向另一个文件。

文件类型信息包含在 stat 结构的 st_mode 成员中。可以通过下表的宏确定文件类型。这些宏的参数都是 stat 结构中的 st_mode 成员。

宏 文件类型
S_ISREG() 普通文件
S_ISDIR() 目录文件
S_ISCHR() 字符特殊文件
S_ISBLK() 块特殊文件
S_ISFIFO() 管道或FIFO
S_ISLNK() 符号链接
S_ISSOCK() 套接字

补充：S_ISXXX 宏的实现
早期的 UNIX 系统版本并不提供 S_ISXXX 宏，于是就需要将 st_mode 与屏蔽字 S_IFMT 进行逻辑与运算，然后再与相应的 S_IFXXX 常量进行比较。这些常量与屏蔽字大多定义在文件 <sys/stat.h> 中。事实上，S_ISXXX 宏的定义就是对上述过程的一个封装，例如 S_ISDIR 宏定义为：
#define S_ISDIR(mode) (((mode) & S_IFMT) == S_IFDIR)

i 节点

i 节点是固定长度的记录项，它包含文件的大部分信息。

i 节点包含了大多数与文件有关的信息：文件类型、文件访问权限位、文件长度和指向该文件所占用的数据块的指针数等等。stat 结构中的大多数信息都取自 i 节点。只有两项数据存放在目录项中：文件名和 i 节点编号。i 节点编号的数据类型是 ino_t 。

文件时间

Linux 对每个文件保持有三个时间字段。

字段 说明 例子 ls(1) 选项
st_atime 文件数据的最后访问时间 read -u
st_mtime 文件数据的最后修改时间 write 默认
st_ctime i 节点状态的最后更改时间 chmod, chown -c

其中值得注意的是，st_mtime 与 st_ctime 的区别，st_mtime 指的是文件内容的最后一次修改时间。 st_ctime 指的是 i 节点最后次的修改时间，因为 i 节点存储着许多与文件属性（如：文件长度、文件权限、文件类型等）有关的信息，所以如果对文件这些属性进行更改将影响 i 节点的修改时间。

符号链接

符号链接是指向一个的文件的间接指针，而硬链接是直接指向文件的 i 节点。引入符号链接的目的是为了打破硬链接的一些限制：

  • 硬链接通常要求链接和文件位于同一个文件系统之中
  • 只有超级用户才能创建指向目录的硬链接

而对符号链接以及它指向何种对象并无任何文件系统限制，任何用户都可以创建指向目录的符号链接。符号链接一般用于将一个文件或整个目录结构移到系统中的另一位置。

Linux 文件内容删除

只有当链接计数达到 0 时，一个文件的内容才可以被删除。另外如果有进程打开了该文件，其内容也不能删除。
因此，关闭一个文件时，内核首先检查打开该文件的进程数，如果为 0，内核再检查该文件的链接数，如果链接数也为 0 ，则删除该文件内容。

重要函数

stat, fstat, lstat

一旦给出 pathname ， stat 函数就返回与此命名文件有关的信息结构。 fstat 函数获取已在描述符 filedes 上打开的文件的有关信息。 lstat 函数类似于 stat ，但是当命名文件是一个符号链接（下文将更详细地说明符号链接）时， lstat 返回该符号链接的有关信息，而不是由该符号链接引用文件的信息。

chmod, fchmod 更改文件访问权限

chmod 在指定的文件上进行操作，而 fchmod 则对已经打开的文件进行操作。

chown, fchown, lchown 用于更改文件的用户 ID 和组 ID

owner 或者 group 参数中任意一个是 -1，则对应的 ID 不变。

在符号链接的情况下， lchown 更改符号链接本身的所有者，而不是该符号链接所指向的文件，除此之外，这三个函数操作相似。

truncate, ftruncate 进行文件截短

这两个函数将把现有的文件长度截短为 length 字节。如果该文件以前的长度大于 length ，则超过 length 以外的数据就不能再访问。若以前的长度小于 length ，则其效果与具体的系统有关。

link, unlink, remove, rename

任何一个文件都允许多个目录项指向其 i 节点。可以通过 link 函数创建一个指向现有文件的链接。

此函数创建一个新目录项 newpath ，它引用现有的文件 existingpath 。如若 newpath 已经存在，则返回出错。而且只创建 newpath 中的最后一个分量，路径中的其他部分应当已经存在。

可以使用 unlink 来删除一个现有的目录项。

这个函数可以删除目录项，并将由 pathname 所引用文件的链接数减 1。如果还有指向该文件的其他链接，则仍可以通过其他链接访问该文件的数据。
如果 pathname 是符号链接，那么 unlink 删除该符号链接，而不会删除由该链接所引用的文件。给出符号链接名的情况下，没有任何函数能删除由该链接所引用的文件。

也可以使用 remove 函数来解除对一个文件或者目录的链接。对于文件， remove 的功能与 unlink 相同。对于目录， remove 的功能与 rmdir 相同。

rename 函数对文件或目录名进行更改。

symlink, readlink 函数

symlink 函数用于创建符号链接。创建一个指向 actualpath 的新目录项 sympath ，在创建符号链接时不要求 actualpath 已经存在。

这个函数组合 open, read, close 函数的所有操作。在 buf 中返回的符号链接的内容不以 null 字符终止。

mkdir, rmdir 目录操作

此函数创建一个新的空目录。其中， . 和 .. 目录项是自动创建的。所指定的文件访问权限 mode 由进程的文件模式创建屏蔽字修改。

用 rmdir 函数可以删除一个空目录。空目录只包含 . 和 .. 两个目录项。

读目录

对某个目录具有访问权限的任一用户都可以读该目录，但是，为了防止文件系统产生混乱，只有内核才能写目录。

目录的实际格式依赖于 UNIX 系统，特别是其文件系统的具体设计和实现。这就意味着读目录的程序与系统有关。为了简化这种情况，UNIX 现在包含了一套与读目录有关的例程。很多实现阻止应用程序使用 read 函数读取目录的内容，从而进一步将应用程序与目录格式中与现实相关的细节隔离开。

chdir, fchdir, getcwd 文件目录操作

每个进程都有一个当前工作目录，此目录是搜索所有相对路径名的起点。当用户登录到 UNIX 系统时，其当前工作目录通常是口令文件（/etc/passwd）中该用户登录项的第 6 个字段——用户的起始目录（home directory）。当前工作目录是进程的一个属性，起始目录则是登录名的一个属性。

另外 fchdir 函数为我们提供了一个更加便捷的方法。在更换到文件系统中的不同位置前，无需调用 getcwd 函数，而是使用 open 打开当前工作目录，然后保存文件描述符。当希望回到工作目录时，只要简单地将文件描述符传递给 fchdir 。

补充

  1. 每个进程都有一个根目录用于解析绝对路径名，可以通过 chroot 函数改根目录。 chroot 函数用于辅助因特网文件传输程序（FTP）中的安全性。系统中没有账号的用户（匿名FTP）放在一个单独的目录下，利用 chroot 将此目录当做新的根目录就可以阻止用户访问此目录以外的文件。 chroot 也用于在另一台机器上构造以文件系统层次结构的一个副本，然后修改此副本，但不更改原来的文件系统。这可用于测试新软件包的安装。 chroot 只能由超级用户使用，一旦更改了一个进程的 root ，该进程及其后代进程就再也不能恢复至原先的 root 。

  2. 对 cpio 来说，既可以改变文件的访问时间（st_atime）和修改时间（st_mtime），也可以都不改变。 cpio 的 -a 选项可以在读文件后重新设置文件的存取时间，改变文件的存取时间。另一方面， -m 将文件的修改时间和存取时间保存为归档时的值。
    对 tar 来说，在抽取文件时，其默认方式是复原归档时的修改时间，但是 -m 选择项将修改时间设置为抽取文件时的时间。无论 tar 在何种情况，文件的存取时间均是抽取文件时的时间。
    由于不能修改状态改变时间（utime 也只能改变访问时间和修改时间，修改状态时间 st_ctime 字段被自动更新），所以没有将其保存在文档上。
