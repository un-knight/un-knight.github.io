STDIN_FILENO 与 stdin 的区别

发表于 2016-11-15   |   分类于 Linux   |     |   阅读次数

文件描述符

文件描述符 STDIN_FILENO、STDOUT_FILENO 和 STDERR_FILENO 是 POSIX 定义的三个整型常量，用来代替 0、1、2，分别代表了标准输入、标准输出和标准错误。位于头文件 unistd.h 。

区别

  1. 数据类型不一致：
    stdin 类型为FILE* ，STDIN_FILENO 类型为 int 。
    使用 stdin 的函数主要有：fread、fwrite、fclose 等，基本上都以f开头；使用 STDIN_FILENO 的函数有：read、write、close 等。

  2. stdin 属于标准 I/O ，高级的输入输出函数。在。STDIN_FILENO是文件描述符，是非负整数，一般定义为 0、1、 2，属于没有缓冲的 I/O ，直接调用系统调用。

  4. 层次不一样。stdin 属于标准库处理的输入流，其声明为 FILE 型的指针，对应的函数前面都有 f 开头，如 fopen/fread/fwrite/fclose 标准库调用等 。STDIN_FILENO 属于系统 API 接口库，其声明为 int 型，是一个打开文件句柄，对应的函数主要包括 open/read/write/close 等系统级调用。

  5. 操作系统一级提供的文件 API 都是以文件描述符来表示打开的文件。STDIN_FILENO 就是标准输入设备（一般是键盘）的文件描述符。
