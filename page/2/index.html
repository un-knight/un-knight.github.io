第四章：文件和目录

发表于 2016-11-24   |   分类于 Linux   |     |   阅读次数

重要概念

文件类型

UNIX 系统的文件类型包括以下几种：

  1. 普通文件 (regular file) 。这种文件包含了某种形式的数据。至于这种数据是文本还是二进制数据对于 UNIX 内核而言并无区别。对普通文件内容的解释由该文件的应用程序进行。

补充：二进制可执行文件是一个值得注意的例外。为了执行程序，内核必须理解其格式。所有二进制可执行文件都遵循一种格式，这种格式使内核能够确定程序文本和数据的加载位置。

  1. 目录文件 (directory file) 。这种文件包含了其他文件的名字以及指向与这些文件有关信息的指针。对一个目录文件具有读权限的任一进程都可以读该目录的内容，但只有内核可以直接写目录文件。进程必须使用特定的函数才能更改目录。
  2. 块特殊文件 (block special file) 。这种文件类型提供对设备（例如磁盘）带缓冲的访问，每次访问以固定长度为单位进行。
  3. 字符特殊文件 (character special file) 。这种文件类型提供对设备不带缓冲的访问，每次访问长度可变。系统中的所有设备要么是字符特殊文件，要么是块特殊文件。
  4. FIFO 。这种类型文件用于进程间通信，有时也将其称为命令管道 (named pipe) 。
  5. 套接字 (socket) 。这种文件类型用于进程间的网络通信。套接字也可用于在一台宿主机上进程之间的非网络通信。
  6. 符号链接 (symbolic link) 。这种文件类型指向另一个文件。

第三章：文件 I/O

发表于 2016-11-23   |   分类于 Linux   |     |   阅读次数

专业名词

  1. 不带缓冲：指的是每个 read 和 write 都调用内核中的一个系统调用。
    PS：这里的“不带缓冲”并不说 read 与 write 函数就不带缓冲机制。事实上，所有的磁盘 I/O 都要经过内核的块缓存器，唯一例外的是对原始磁盘设备的 I/O ，但是我们不考虑。既然 read 或 write 的数据都要被内核缓存，那么术语“不带缓存的 I/O”指的是在用户的进程中对这两个函数不会自动缓存，每次 read 或者 write 就要进行一次系统调用。

  2. 文件描述符：文件描述符是一个非负整数，当使用 create 或者 open 函数创建或者打开一个文件的时候，内核向进程返回一个文件描述符。对内核而言，所有对打开的文件都通过文件描述符进行引用操作。

  3. 延迟写：当要将数据写入文件时，内核通常先将该数据复制到其中一个缓冲区块中，等待该缓冲区块写满或者当内核需要重用该缓冲区以便存放其他磁盘块数据时，再将该缓冲排入输入队列，然后待其到达队首时，才进行实际的 I/O 操作。
    优缺点：延迟写虽然减少了磁盘读写次数，但是却降低了文件内容的更新速度，可能导致磁盘上实际文件内容与缓冲区中内容不一致的问题。

STDIN_FILENO 与 stdin 的区别

发表于 2016-11-15   |   分类于 Linux   |     |   阅读次数

文件描述符

文件描述符 STDIN_FILENO、STDOUT_FILENO 和 STDERR_FILENO 是 POSIX 定义的三个整型常量，用来代替 0、1、2，分别代表了标准输入、标准输出和标准错误。位于头文件 unistd.h 。

区别

  1. 数据类型不一致：
    stdin 类型为FILE* ，STDIN_FILENO 类型为 int 。
    使用 stdin 的函数主要有：fread、fwrite、fclose 等，基本上都以f开头；使用 STDIN_FILENO 的函数有：read、write、close 等。

  2. stdin 属于标准 I/O ，高级的输入输出函数。在。STDIN_FILENO是文件描述符，是非负整数，一般定义为 0、1、 2，属于没有缓冲的 I/O ，直接调用系统调用。
